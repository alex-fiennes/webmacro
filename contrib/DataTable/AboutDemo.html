About the DataTable Sample Application

The sample application uses a servlet named TableDemo.java which subclasses WMServlet and a template called TableDemo.wm. It is designed to illustrate most of the features of the ResultSetDataTable.

The demo uses the hsql Database Engine and the Protomatter connection pooling software. Both are pure-Java, open-source products. The data is from the sample database from the Hypersonic SQL demo.

To use the HypersonicSQL sample database - Make sure the database is running in client/server mode ("runServer"). Use the database manager application ("run DatabaseManager") and select from the menu "Option|Insert test data" to create the sample tables.

You can easily substitute any database and/or connection pool that works with JDBC.

To get the demo working on your system - you will either need to install the above packages, in addition to WebMacro, or modify the code to work with your own database. Add the org.webmacro.datatable classes to your CLASSPATH and be sure to adjust the TemplatePath in the WebMacro.properties file to point to the location of TableDemo.wm and the sub-template, TableNav.wm.

The template uses a #use 'text' directive, to escape the JavaScript code, so you may have to modify the syntax of that directive to match your WebMacro version.

Using your own database - You will need to modify the JDBC connection code and the SQL in TableDemo.java, as well as the named column references in the template (e.g., $row.FIRSTNAME), to match your own database.
