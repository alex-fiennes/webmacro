Step 3/4: Bind actions to action methods of components via configuration file (snippet follows): 

<application repository="c:/progra~1/ActionServlet/examples/LoginServlet/classes">
   <templates>
      <template name="Login.wm" is-new-session="true"/>
   </templates>

   <components>
      <component class="Authenticator" persistence="application">
         <action name="Login" method="login(String userName, String password)"/>
      </component>
   </components>
</application>

The configuration file has a simple XML format. Each component has defined class and persistence (application/session/request) and one or more action subelements. Here is bound Login action to Authenticator.login(WebContext context, String userName, String password) method. Template with is-new-session attribute set to true is shown upon a new session (when the user first visits the servlet page).

Note: The context variable is always omitted in the action definition.
