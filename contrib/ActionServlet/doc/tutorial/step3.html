Step 3/4: Bind actions to action methods of components in the configuration file called ActionConfig: 

<application>
   <templates>
      <template name="Login.wm" is-new-session="true"/>
   </templates>

   <components>
      <component name="Authenticator" class="Authenticator" persistence="session">
         <action name="Login" method="login(String userName, String password)"/>
      </component>
   </components>
</application>

ActionConfig has a simple XML format. Each component has defined name, class and persistence (one of: application/session/request) and zero or more <action> subelements. Here is bound Login action to Authenticator.login(String userName, String password) method.

The template with an is-new-session attribute set to true is shown upon a new session (when the user first visits the servlet's page).
